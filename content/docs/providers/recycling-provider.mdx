## Usage

The Recycling Provider efficiently handles large lists by progressively loading items as the user scrolls. It uses a render prop pattern to provide the visible items and scroll handling functions.

```tsx
import { RecyclingProvider } from "@/components/providers/recycling-provider";

export default function Example() {
  const items = Array.from({ length: 10000 }, (_, i) => i);

  return (
    <RecyclingProvider items={items} itemsPerPage={50}>
      {({ visibleItems, scrollRef, onScroll }) => (
        <div ref={scrollRef} onScroll={onScroll} className="h-[600px] overflow-auto">
          {visibleItems.map((item) => (
            <div key={item}>{item}</div>
          ))}
        </div>
      )}
    </RecyclingProvider>
  );
}
```

## How It Works

1. **Initial Load**: Loads the first `itemsPerPage` items
2. **Scroll Detection**: Monitors scroll position to detect when user approaches the bottom
3. **Progressive Loading**: Automatically loads more items when needed
4. **Auto-fill**: On mount, checks if the container needs more items to enable scrolling

## API Reference

### RecyclingProvider Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `T[]` | - | The full array of items to render |
| `itemsPerPage` | `number` | - | Number of items to load per batch |
| `children` | `(props: RenderProps) => ReactNode` | - | Render function receiving visible items and scroll handlers |

### Render Props

| Property | Type | Description |
|----------|------|-------------|
| `visibleItems` | `T[]` | Currently visible items (loaded so far) |
| `scrollRef` | `React.RefObject<HTMLDivElement \| null>` | Ref to attach to scrollable container |
| `onScroll` | `() => void` | Scroll handler to attach to scrollable container |
